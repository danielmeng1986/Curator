# 06 · Roadmap

> Curator Development Roadmap

---

# Overview

Curator is designed as a long-term project.

Rather than focusing on isolated features, development is organized into a series of capability milestones.

Each milestone builds upon the previous one while preserving the project's architectural principles.

The roadmap emphasizes sustainable evolution over rapid feature accumulation.

---

# Development Principles

The roadmap follows several guiding principles.

- Build stable foundations before adding intelligence.
- Preserve architectural consistency.
- Prefer incremental progress.
- Keep every milestone independently valuable.
- Ensure each stage leaves the project in a usable state.

---

# Phase 1 — Foundation

**Goal**

Build a reliable platform for managing digital assets.

### Objectives

- Project architecture
- SQLite database
- Archive scanning
- Normalize pipeline
- Workspace database
- Basic Web UI
- Validation workflow
- Rename Plan
- Commit workflow

### Result

Curator becomes capable of organizing and validating digital assets safely.

---

# Phase 2 — Knowledge Management

**Goal**

Transform file metadata into structured knowledge.

### Objectives

- Studio management
- Album management
- Model management
- Stable identifiers
- Relationships
- Metadata editing
- Annotation support
- Search capabilities

### Result

Curator evolves from a file organizer into a knowledge management system.

---

# Phase 3 — AI Assistance

**Goal**

Introduce AI as a collaborative assistant.

### Objectives

- Face recognition
- Object recognition
- Image captioning
- Album naming suggestions
- Duplicate detection
- Semantic tagging
- Similarity search
- Metadata completion

### Result

AI begins enriching knowledge while remaining under human supervision.

---

# Phase 4 — Intelligent Knowledge

**Goal**

Enable AI to understand relationships rather than individual assets.

### Objectives

- Vector embeddings
- Knowledge graph
- Semantic search
- Relationship inference
- Collection recommendations
- Timeline reconstruction
- Cross-album analysis

### Result

Curator begins understanding the archive instead of merely describing it.

---

# Phase 5 — Personal Intelligence

**Goal**

Build an AI assistant specialized for the user's archive.

### Objectives

- Conversational search
- Context-aware recommendations
- Personalized organization
- Preference learning
- Memory-assisted workflows
- Intelligent planning

### Result

Curator becomes an intelligent companion rather than a management tool.

---

# Phase 6 — Knowledge Platform

**Goal**

Expand beyond media management.

Potential asset types include:

- documents
- books
- notes
- music
- source code
- AI-generated content
- research materials

### Result

Curator becomes a unified personal knowledge platform.

---

# Long-Term Vision

The long-term direction of Curator is to build an ecosystem where:

- digital assets
- structured metadata
- AI understanding
- human knowledge

coexist inside a unified architecture.

The system should continuously accumulate knowledge without losing transparency or maintainability.

# Success Criteria

Curator should eventually provide:

- reproducible workflows
- structured knowledge
- understandable architecture
- maintainable code
- explainable AI
- long-term scalability

Success is measured by maintainability rather than feature count.

---

# Non-Goals

Curator does not aim to become:

- a media player
- a cloud storage provider
- a replacement for the operating system
- a fully autonomous AI

Its purpose is to assist humans in preserving and understanding their digital assets.

---

# Roadmap Evolution

The roadmap is expected to evolve.

New milestones may appear.

Existing milestones may change.

However, every change should remain consistent with the project's architectural principles and long-term vision.

The roadmap should describe direction rather than implementation details.

---

# Summary

Curator evolves through successive stages of capability.

The project begins with reliable asset management, progresses toward structured knowledge, introduces AI assistance, and ultimately aims to become a long-term personal knowledge platform.

Each phase should strengthen the existing architecture rather than replace it.

---

> Curator is built one capability at a time.

> Every milestone should make the system more understandable, more maintainable, and more intelligent than before.
